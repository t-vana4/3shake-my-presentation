# 登壇資料作成ガイドライン

## 基本思想

スライドは「読ませる」ものではなく「見せる」もの。
登壇で最も大事なのは自分の話を聞いてもらうこと。スライドはそのメッセージを伝える道具である。

## 制作フロー

デザインはコンテンツの設計から始まる。以下の順序で進める。

1. **コンテンツ** - 伝えたいことの整理
2. **レイアウト** - 情報の配置を整理
3. **ビジュアル** - 見た目の整理

## コンテンツ設計

### 事前に決めること

- **目的とターゲット** を明確にする
- **時間制約** から逆算して各パートの時間配分を決める（例: 導入5分 → Part1 10分 → Part2 15分 → Part3 10分 → まとめ5分）
- **話さないこと** を明確にする。情報を削ることでスライドがはっきりし中身がわかりやすくなる

### コンテンツの流れ

- 話には流れがある。流れを止めないことが大事
- テーマが切り替わるときは、前の内容とどう接続するのかを伝える（つなぎを入れる）

### スライドの文章5原則

1. **箇条書きを使わない**
2. **タイトル（一番大きいテキスト）で結論をはっきり伝える** - 1スライド1メッセージ
3. **1スライドで5行以上の文章を書かない**
4. **文章は極力改行させないくらい短い文章で書く**
5. **マークダウン的な階層構造を持ち込まない** - スライドは横にスライドしていくものなので階層をイメージできない。フラットな構造で話を組み立てる

### ページ分割

- 1スライドで長く話してしまうのはNG
- ページを分割して、テンポよく話が進むようにする
- 1つのテーマで複数スライド使っても問題ない。スライド＝話の最小単位を意識する

### 無駄を省く

- 無味乾燥なスライドを挟まない。流れが悪くなるし、話の印象が弱くなる
- 余計な笑いを取りに行こうとしない

### 生成AI活用時の注意

- 生成AIに要約をさせると抑揚のない資料になりやすい。重み付けは人間がやる
- 生成AIを活用する場合はレイアウトを整える部分にフォーカスするのが良い

## デザイン原則

以下の色・フォント・基本レイアウトは `themes/3shake` テーマに定義されている。テーマの値を前提として記載する。

### テーマカラーパレット（themes/3shake 準拠）

- **ベースカラー（濃紺）**: `--color-navy: #011121` - タイトル、見出し、coverやsectionの背景
- **本文テキスト**: `--color-dark-text: #2d2d2d`
- **グレー**: `--color-gray: #6b6b6b` - section背景、補助テキスト
- **ライト背景**: `--color-light-bg: #f5f6f8` - スライド全体の背景色
- **アクセントカラー**:
  - グリーン: `--color-green: #2aa87f` - 主要アクセント（メトリクス、ステップ、ラベルなど）
  - ブルー: `--color-blue: #2b9bc2` - リンク、補助アクセント
  - イエロー: `--color-yellow: #ddac26` - 補助アクセント
- **色はベースカラー + アクセント1色で十分** - フォントサイズや太字で差をつければ多色は不要。アクセント使用は1スライドに1〜2箇所に絞る

### タイポグラフィ（themes/3shake 準拠）

- **フォント**: Noto Sans JP, Hiragino Kaku Gothic ProN, Hiragino Sans, BIZ UDPGothic, sans-serif
- **テキストの3段階ジャンプ率で「何を見るべきか」を瞬時に伝える**:
  - **h1（タイトル）**: 34px / 太字700 - コンテンツスライドの結論・テーマ。sectionスライドでは60px
  - **h2（サブタイトル）**: 28px / 太字700 - タイトルの補足
  - **本文**: 22px / 行間1.8 - 補足情報。読ませるのではなく見せる
- **キーメッセージ以外はわざと小さくする** - 文章は読まなくても良いというアピールをする
- **コントラスト比は 9:1 を目指す** - プロジェクター投影で物理的に見えなくなるのを防ぐ

### 基本レイアウト（themes/3shake 準拠）

テーマの default レイアウトは以下の構造を持つ:

- **背景**: 上部80pxが白、それ以下が `--color-light-bg` のグラデーション
- **パディング**: `100px 80px 60px 40px`（上 右 下 左）
- **h1**: absolute配置でスライド上部に固定
- **ロゴ**: 右上に120px幅で配置
- **ページ番号**: 右下に配置

この基本構造の上に、以下の原則を適用する:

- **中央揃えは使わない、左端を揃える** - 枠線を意識して、そこにあわせにいくイメージ
- **空白を恐れない** - 空白ができても、無理に埋めなくて良い
- **黒背景は基本使わない** - プロジェクターは黒色が多いほど画面全体の明るさが低くなる。特にコードブロックに注意

### レイアウトの具体パターン（理想スライドから抽出）

テーマの基本レイアウト上で、以下のパターンを活用する。

#### 全体構造（ヘッダー + コンテンツエリア）

スライドは「ヘッダーエリア（上部1/3）」と「コンテンツエリア（下部2/3）」に大きく分かれる。

- **ヘッダーエリア**
  - 1行目: h1タイトル（テーマ定義の34px太字、結論・テーマを伝える）
  - 2行目: サブタイトル/キーメッセージ（h2の28pxまたは本文サイズ）。タイトルの補足説明を1〜2行で書く
  - ヘッダーとコンテンツの間には十分な余白を取る
- **コンテンツエリア**
  - カード・図解・フロー図などをここに配置する
  - コンテンツエリアの下部にも余白を残す

#### カードベースのコンテンツ配置

コンテンツは直接スライド上に文字を置くのではなく、**テーマの `.box` クラスや角丸のカード（コンテナ）** に入れて配置する。

- **カードの背景**: `--color-light-bg`（#f5f6f8）やグラデーション。テーマの `.box` `.box-green` `.box-blue` 等を活用
- **カードの角丸**: テーマ定義の 8px
- **カード内部のパディング**: テーマの `.box` は 20px 28px
- **カード内のテキスト**: 左揃え、コンパクト
- **カード同士の間隔**: テーマの `.flex` `.gap-lg`（32px）等を活用

#### 代表的なカードレイアウトパターン

1. **2カラムカード**: 左右にカードを並べて対比・比較を表現する（最も頻出）
   - テーマの `.flex` で横並び、各カード `.box` で囲む
   - 左右のカードの幅は均等、または内容量に応じて調整
2. **4カードグリッド（2x2）**: 4つのポイントを整理して並べる
   - テーマの `.step` で番号付きバッジ、`.box` でカード化
   - カードの下に2〜3行の補足テキスト
3. **フロー図**: テーマの `.flow-box` と矢印（→）でプロセスや進行を示す
   - `.flow-box-green` `.flow-box-blue` `.flow-box-gray` で色分け
   - ボックスの下にラベル（小さいテキスト）で補足
4. **タイムライン/バー図**: テーマの `horizontal-bar` `stacked-bar` レイアウトを活用
   - バーの色の濃淡で役割を区別
   - 下部に凡例を配置

#### テーマのユーティリティクラス活用

スライド作成時は以下のテーマ定義クラスを積極的に使う:

- **配置**: `.flex` `.flex-center` `.flex-between` `.flex-col` `.flex-wrap`
- **間隔**: `.gap-sm`(8px) / デフォルト(16px) / `.gap-lg`(32px) / `.gap-xl`(48px)
- **装飾**: `.box` `.box-{color}` `.box-{color}-outline` / `.label` `.label-{color}` / `.badge` `.badge-{color}`
- **強調**: `.marker-green` `.marker-blue` `.marker-yellow`（背景ハイライト）
- **区切り**: `.divider-h` `.divider-v` + 色バリアント
- **数値表示**: `.metrics` `.metric-num` `.metric-unit` `.metric-label`

### アニメーション

- 中途半端なアニメーションはかえってノイズになる
- アニメーションを入れるならきちんと設計する
- アニメーションではなくて装飾で変化を伝えるという手もある

### 図・グラフ

- 図やグラフは積極的に活用する
- グラフを描かせたいときはデータソースとインフォグラフィック系の参考資料を与える

## デザインフレームワーク（チェックリスト）

スライド作成時に以下を確認する:

1. **目的とターゲット** が明確か
2. **時間制約** を把握し、パートごとの時間配分を決めたか
3. **話さないこと** を決めたか
4. **構成と時間配分（ストーリーの流れ）** を設計したか
5. **コンテンツ**
   - 1スライド1メッセージ（タイトルで結論を言う）
   - 1スライド5行以内の短い文章にする
   - 箇条書き・階層構造（マークダウン的）は極力使わない
   - コンテンツが「つながって」いる
   - 無味乾燥なスライドや不要なアニメーションは排除する
6. **ビジュアル・トンマナ**（themes/3shake 準拠）
   - カラー: ベースカラー（navy #011121）+ アクセント（green #2aa87f / blue #2b9bc2 / yellow #ddac26）から1色
   - タイポグラフィ: h1 34px太字 / h2 28px太字 / 本文 22px の3段階ジャンプ率
   - 左端揃え、余白を恐れない、コントラスト比 9:1、黒背景を避ける
7. **レイアウト**
   - ヘッダー（上部1/3: h1+サブタイトル）とコンテンツ（下部2/3: カード・図解）の2エリア構成
   - コンテンツはテーマの `.box` `.flow-box` 等のカードに入れて配置
   - テーマの `.flex` `.gap-lg` 等のユーティリティクラスを活用
